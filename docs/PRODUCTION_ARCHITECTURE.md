# GigAI Bharat — Production Architecture

**Single toolchain:** Cursor → GitHub → Vercel → Supabase

---

## Repository layout

```
gigai-bharat/                 ← Git root (this repo)
├── apps/
│   ├── marketing/            ← www.bharatgig.live (TanStack Start + Nitro SSR)
│   ├── worker/               ← app.bharatgig.live (React + Vite SPA)
│   └── admin/                ← Internal ops (not public)
├── packages/                 ← Shared types, UI, tsconfig
├── supabase/                 ← Postgres, Auth, Edge Functions, RLS
├── scripts/
│   ├── prepare-worker-vercel-output.cjs
│   └── deploy-vercel.ps1
└── .github/workflows/
    ├── ci.yml
    └── deploy-production.yml
```

**Do not use:** nested `gigai-bharat/` folder, Lovable remix deploys, Cloudflare/Wrangler as primary host, duplicate Vercel projects.

## Deployment flow

1. **Develop** in Cursor (monorepo root)
2. **Commit & push** to `main` on GitHub
3. **GitHub Actions** (`deploy-production.yml`) builds and deploys both apps to Vercel (prebuilt)
4. **Supabase** handles auth, database, edge functions independently

Manual fallback:

```powershell
.\scripts\deploy-vercel.ps1 -Target all -SkipCache
```

---

## Worker SPA routing

All paths on `app.bharatgig.live` rewrite to `index.html`:

- `apps/worker/vercel.json` — Vercel Git deploy (root: `apps/worker`)
- `.vercel/output/config.json` — prebuilt deploy (generated by `prepare-worker-vercel-output.cjs`)

Auth routes: `/auth`, `/auth/callback`, `/oauth/callback`, `/~oauth/initiate`

Post-login: `/dashboard` (or `/onboarding` for new workers)

---

## Marketing SSR routing

Nitro server handles all routes. Static assets served from `.vercel/output/static`.

Key pages: `/`, `/gurukul`, `/shramsetu`, `/investors`, `/ev-infrastructure`, `/smart-hub`

---

## Backend (Supabase)

- **Auth:** Google OAuth via Supabase PKCE → `/auth/callback`
- **Database:** `worker_profiles`, earnings ledger, RLS on all worker tables
- **Edge Functions:** `parse-earning` (OCR pipeline)

Redirect URLs: see [AUTH_PRODUCTION.md](./AUTH_PRODUCTION.md)
